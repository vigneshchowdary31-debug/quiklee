Database Architecture

The storage layer uses a decoupled relational pool designed to ensure data consistency, query protection, and multi-environment adaptability.

Relational Storage

  • PostgreSQL Database: Hosted on a cloud-based server instance with SSL parameters enabled.
  • SQLite Fallback: Local file-based configuration available for offline prototyping environment setup.
  • Connection Pooling: Managed size limitations to allocate resources across concurrent clients safely.

Security & Adapters

  • Parameterized SQL: Binds query placeholders dynamically to shield the database from SQL Injection tags.
  • Execution Adapter: Adapter wraps PG Pool queries to translate MySQL query formats (`?`) into PostgreSQL variables (`$1`).

Relational Schema & Tables

Database entities are fully normalized with relational keys to log transactions, alerts, and supplier updates cleanly.

Core Entities

  • products: Holds SKU values, stock quantities, status badges, and reorder threshold levels.
  • alerts: Logs low-stock or out-of-stock events triggered by updates to product quantities.
  • inventory_history: Automatically records stock logs to display dashboard charts.

Suppliers & Logs

  • suppliers: Contact log indexing name and email details of product suppliers.
  • supplier_orders: Manages supply chains and records purchase request status.
  • sales: Records sales totals and quantities for summary report graphing.

Deployment Architecture

The deployment isolates presentation modules from processing layers, keeping files and database structures decoupled.

Decoupled Hosting

  • Vercel Frontend: Hosts the client-side SPA files on a globally accessible edge CDN.
  • Railway API: Deploys the Node/Express backend container securely behind environment ports.
  • Railway PostgreSQL: Manages the transactional relational storage layer.

Delivery Channels

  • Automated CI/CD: Triggers deployment compilation runs automatically on GitHub commits.
  • Origin Policy: Express CORS header controls allow calls coming only from Vercel origins.
  • Secrets Masking: Protects database links and cryptographic credentials.

Frontend Hosting on Vercel

Vite Compilation & Routing

  • Asset Optimizations: Builds client files into a minified, cached static directory bundle.
  • Rewrite Configurations: Routes all browser URL searches back to `index.html` to support client-side React Router navigation.
  • Deploy Pipeline: Integrates with git to trigger preview builds on pull requests.

vercel.json Config

{
  "version": 2,
  "framework": "vite",
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "rewrites": [
    { 
      "source": "/(.*)", 
      "destination": "/index.html" 
    }
  ]
}

Backend Hosting on Railway

Nixpacks & Docker Containers

  • Runtime Packaging: Nixpacks detects the Node environment and generates container build rules automatically.
  • Restart Policies: Deploys container execution runners with automated crash restart boundaries.
  • Scale Safety: Migrated from SQLite to persistent PostgreSQL to avoid data loss on restarts.

railway.json Config

{
  "$schema": "https://railway.app/...",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "node server.js",
    "restartPolicyType": "ON_FAILURE"
  }
}
